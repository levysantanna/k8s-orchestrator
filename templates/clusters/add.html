Connection Methods

Choose how to connect to your cluster:

1. Kubeconfig Upload

Upload an existing kubeconfig file

  • Standard Kubernetes authentication
  • Works with any cluster
  • Optionally add SSH for metrics
2. SSH Connection

Connect using SSH credentials only

  • Auto-retrieves kubeconfig via SSH
  • No manual file download needed
  • Requires SSH access to master node
  • Supports private key or password auth
SSH Requirements

Your SSH user must have:

  • Read access to kubeconfig
  • Sudo privileges (for k3s: root or in k3s group)
